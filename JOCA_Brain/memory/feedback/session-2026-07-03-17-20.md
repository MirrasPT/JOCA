**Categoria:** tool-reliability
**Severidade:** medium
**Descricao:** O MCP `playwright` (`mcp__playwright__browser_navigate`) recusa `file:///` directamente ("Access to 'file:' protocol is blocked"), inviabilizando medir/renderizar ficheiros HTML locais sem servidor. Contorno usado: `python -m http.server <porta>` em background na pasta do ficheiro, navegar via `http://127.0.0.1:<porta>/ficheiro.html`.
**Componente afectado:** MCP playwright (global).
**Fix sugerido:** Documentar este contorno em `.claude/reference/workflows-and-tooling.md` (gotchas de ambiente) para não ser redescoberto por sessão.

**Categoria:** missing-skill
**Severidade:** low
**Descricao:** Não existe skill/agente para "exportar HTML para PDF de página única fielmente ao layout" — foi preciso descobrir manualmente que o Chrome headless (`--print-to-pdf`) usa tamanho Letter por omissão e corta conteúdo mais alto para 2ª página, e que a correcção é embutir `@media print { @page { size: <w>px <h>px; margin:0 } }` com a altura medida via Playwright (`document.body.scrollHeight`).
**Componente afectado:** nenhuma skill actual cobre "html-to-pdf single-page export".
**Fix sugerido:** Considerar uma skill leve `html-to-pdf` (ou adenda à `frontend`) com esta receita: medir altura via Playwright a viewport fixa → injectar `@page` custom → `chrome.exe --headless=new --print-to-pdf`. Reutilizável para qualquer documento HTML (orçamentos, relatórios, fichas técnicas) que precise de export PDF fiel.
