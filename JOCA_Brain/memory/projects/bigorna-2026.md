- **Performance (fix 2026-05-31):** páginas estavam a ~2,5s. Causa: web servido por `php artisan serve` (SAPI **CLI**) com `opcache.enable_cli=0` → recompilava tudo por request, agravado por bind-mount Windows + `D:\Mega` (MEGAsync activo). Fix persistente: `docker/php/99-joca-perf.ini` (opcache CLI on, `validate_timestamps=0`, xdebug off) montado via volume no `compose.yaml`; sessão+cache → Redis. Resultado ~0,5s. Projecto fica no Mega (decisão do Renato).
- **GOTCHA opcache `validate_timestamps=0`:** depois de editar **código PHP**, o `artisan`/testes correm código **STALE** (opcache CLI ligado). Para ver mudanças: ou `docker compose restart laravel.test` (web) ou correr com bypass: `docker compose exec -T laravel.test php -d opcache.enable_cli=0 artisan <test|migrate>`. Sem isto, edições "não fazem nada".
- **Worker + scheduler (desde 2026-05-31):** serviços compose `queue.worker` (`queue:work database --queue=mail,default`) e `scheduler` (`schedule:work`). Arrancar: `docker compose up -d queue.worker scheduler`. **Mesma gotcha de opcache:** após editar Job/Listener/Mailable/Action/Command, `docker compose restart queue.worker scheduler` (senão processam código stale). Emails só SAEM quando o token SMTP ZeptoMail estiver preenchido (até lá `mail.default`=log/array, mas `EmailLog` regista na mesma).
- **GOTCHA mailable base:** `BigornaMailable` faz só `onQueue('mail')` (NÃO `onConnection('database')`) → em testes (`QUEUE_CONNECTION=sync`) corre inline e os listeners de mail disparam. Testes de `EmailLog` usam o array transport (`MAIL_MAILER=array` no phpunit), **nunca `Mail::fake()`** (saltaria MessageSending/MessageSent).
- **FRONTEND DEV (Vite) — correr no HOST, não no container (gotcha 2026-06-23):** o container `laravel.test` mapeia `5173:5173`, mas o `npm install` foi feito no HOST (Windows) → `node_modules` tem binários nativos Windows (`@rolldown/binding-win32-x64-msvc`).
