- **Gemma 4 como gerador de prompt:** nó **`TextGenerate`** (built-in, `comfy_extras/nodes_textgen.py`, alias "gemma") — NÃO precisa de KJNodes. Toma `clip`(LLM) + `prompt`(STRING) → `generated_text`(STRING). `sampling_mode:"off"` = greedy (determinístico, JSON fiável); `"on"` expõe temperature/top_k/top_p.
- **Carregar o gemma4 como LLM:** `CLIPLoader(clip_name:"gemma4_e4b_it_fp8_scaled.safetensors", type:"lumina2")`. O `type` da lista não tem "gemma4" mas o ComfyUI **auto-detecta pelos pesos** (`sd.py:1538`, `te_model in GEMMA_4_*`). `lumina2` confirmado por teste (output "Hello"). `Gemma4Base` herda `BaseGenerate` → tem `.generate()`.
- **`max_length` = tokens NOVOS, não total** (`llama.py:879` `embeds.shape[1]+max_length`). System prompt de ~28k chars não conta; 2048 chega para a JSON (~1k tokens).
- **Wiring:** `212`(template montado) → `TextGenerate.prompt`; `CLIPLoader gemma`→`.clip`; `TextGenerate`→`CLIPTextEncode(186).text` (substitui o JSON hardcoded).
- **Entregável final: `Ideogram4_t2i_gemma.json`** (UI-format, subgrafos preservados). Patch via `D:\_Comfyui\patch_subgraph_wf.js`: adiciona 2 nós top-level (`CLIPLoader` gemma + `TextGenerate`) e liga `134`(builder)→`TextGenerate`→`98.text`. NÃO mexe nos internos dos subgrafos. Dentro do subgrafo `98`, o input `text` já liga ao `CLIPTextEncode` (o JSON hardcoded é só fallback do widget — o link ganha). Carregar ESTE no browser (Workflows).
